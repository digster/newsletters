Choosing the correct representation plays an important role in solving problems that are modeled using graphs. And sometimes, the best approach is to use a combination of them. Some takeaways. We have three main ways of representing graphs in a computer program: edge lists, adjacency matrices, and adjacency lists. Edge lists are useful for implementing algorithms that process the edges of the graph in a given order. Adjacency matrices are a very visual way of representing graphs. It allows for checking the existence of edges in constant time, but they always use O(n²) memory for a graph of n nodes. Adjacency lists are the most common way of representing graphs and the preferred representation for most of the classic graph algorithms we will be exploring.
